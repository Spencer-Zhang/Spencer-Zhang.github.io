About Classes

May 23, 2014

Classes are the basic unit of object-oriented programming. Each class represent a type of object, and encapsulates the variables and methods needed to emulate an object within a computer program.

Classes contain variables, which allows for variability in between different class instances. For example, in the below program, two instances are created of the Dog class. Although both dog1 and dog2 are of the Dog class, the use of variables allows them to be given different names.

class Dog
 attr_accessor:name
end

dog1 = Dog.new()
dog2 = Dog.new()
dog1.name = "Spot"
dog2.name = "Sparky"

Classes also define the actions that an object can perform. The actions that can be performed by an object are defined within methods.

class Dog
 def bark
  puts "Bark!"
 end
end

dog1.bark # prints "Bark!" to the terminal

An important property of Classes is that they can inherit properties from a parent class. For example, both Dogs and Cats can perform the actions "eat" and "sleep", but it is redundant to write eat() and sleep() methods for both classes. Moreover, if we were to add a Fish and Bird class in the future, we would need to redefine those methods for each of the new classes. Instead, it is more efficient to define these methods for an Animal class, and have the Dog and Cat classes inherit these methods from the Animal class.
